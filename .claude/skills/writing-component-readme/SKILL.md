# Structure of a Component README

When writing a README for a Systemface component, it's important to include the following sections:

- 1. Title and Description: Start with the name of the component and a brief description of its purpose and functionality.
- 2. Code Example: Provide a simple code snippet demonstrating how to use the component in a React application.
- 3. Props: List the props that the component accepts, including their types, default values, and descriptions.
- 4. Usage Examples: Show different ways to use the component, including variations in props and styling.
- 5. CSS Handles: If applicable, list the CSS handles that can be used to customize the component's styling, along with examples of how to override them with custom CSS.
- 6. Additional Resources: Link to the component's Storybook page or any relevant documentation for further reference.

Example: See the [Button README](../../../src/components/atoms/Button/README.md) for a well-structured example of a component README.
